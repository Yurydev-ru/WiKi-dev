# Шаблон Pull Request

- Файл: .github/PULL_REQUEST_TEMPLATE.md 
- Цель: дать ревьюеру контекст, чек‑лист и критерии приёма.

```markdown
## Описание
Кратко опиши, что сделано и зачем.

## Тип изменений
- [ ] Bugfix
- [ ] Feature
- [ ] Refactor
- [ ] Chore
- [ ] Docs

## Как протестировать
1. Шаг 1: что сделать локально
2. Шаг 2: команды для запуска (например `npm run dev`)
3. Ожидаемое поведение

## Скриншоты / GIF
(если есть визуальные изменения)

## Чеклист
- [ ] Код форматирован и линт пройден
- [ ] Юнит/интеграционные тесты добавлены или обновлены
- [ ] Документация обновлена (если нужно)
- [ ] Нет конфликтов с `develop`

## Связанные задачи / issue
Closes #<номер> (если применимо)

```
### Совет: добавь шаблон в репо — GitHub предложит его при создании PR.

# Шаблон коммит сообщений
`Стиль:` 
> Адаптированный Conventional Commits — коротко, машинно‑читаемо, удобно для changelog.
`Формат:`
```git
<type>(<scope>): <short summary>
Optional longer description (wrap at ~72 chars).
```
### Типы:
feat — новая функциональность
fix — исправление бага
chore — вспомогательные задачи, обновления зависимостей
refactor — рефакторинг без изменения поведения
docs — документация
style — форматирование, стили, без логики
test — добавление/исправление тестов

### Примеры:
```git
feat(ui): add collapsible sidebar
Добавил компонент Sidebar с поддержкой collapse/expand.
```

```git
fix(nav): correct active link detection
Исправлена логика определения активного пункта меню.
```
### Правила:
- Первая строка до 72 символов.
- Пустая строка между заголовком и телом.
- В теле — причина и краткое описание решения.
- При закрытии issue: fixes #123 в теле коммита.
